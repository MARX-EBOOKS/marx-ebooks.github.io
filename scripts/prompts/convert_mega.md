按照分析版面—编写 HTML 代码—对照原 PDF 页面审校代码的步骤推理，确认无误后输出。

# 分析版面与编写代码
- 1. 识别排版信息时可根据语义联系进行分析，在排版与语义明显不匹配时应及时重新对照图片，据图转换。
- 2. 页眉转为 title。如无页眉，则用本页上下均有横线的最高一级标题作为 title。不要把页眉输出到正文 body 部分，同时注意应把最高一级标题正确识别为 h1，照样输出到 body 中，不要当成页眉。同时识别在相邻页分布的同一段落，作为一个 <p> 标签输出。页脚除脚注外一律抛弃，即抛弃页码等。
- 3. 多余的连字符号也一并删除，词组间空格也须正确加回，根据词组完整性判断是否保留连字符、如何加回空格。如连字符后有空格则保留连字符。注意识别并过滤在侧边的每行行号，结合语义正确合并每段的各行，不要把行号识别成文段内容。
- 4. 正确识别页码签，格式如 ||xxx|、|xxx|、|[xx]| 等，这代表原登载处的某某页。
- 5. 正确识别多种格式的作者脚注，并将正文中作者的脚注标记与每页页脚注释前的编号转换为相互可跳转的双向链接，确保点击正文上标可跳转至对应脚注，在脚注栏点击脚注编号可返回原文位置。按以下规则转换：
   - 作者本人的注解，即 Marx 、Engels 的注解上标，按照识别阶段制定的编号及id输出，编号文本保留原样，Marx的注解格式为：<sup><a id="ZMxx" href="#Mxx">原编号文本</a></sup>，Engles的注解格式为：<sup><a id="ZExx" href="#Exx">原编号文本</a></sup>，不确定作者的上标格式为：<sup><a id="ZFxx" href="#Fxx">原编号文本</a></sup>；
   - 页脚脚注内容集中在body的末尾输出，放在<aside>内。每条脚注分段排列，输出格式为：<p><a id="Mxx/Exx" href="#ZMxx/ZEXX">xx</a> ...</p>，不确定作者用 <p><a id="Fxx" href="#ZFXX">xx</a> ...</p> 输出，有多段则另起一个 <p> 标签，有表格引文等其他格式也务必正确转换、还原。
- 6. 正确识别并转换斜体、靠右、居中、引文、整行缩进等排版，遵照以下要求：
  - 作者引文用 <blockquote> 输出；务必准确识别首行缩进情况，首行有缩进两字符的普通段落转换为纯段落 <p> 或 <blockquote> 标签，首行无缩进的段落务必输出为 <p style="text-indent: 0;"> 或 <blockquote style="text-indent: 0;"> ，不要遗漏；
  - 仅输出body和title，不要输出多余内容，不要把页眉当做标题；
  - 粗体、斜体、靠右、居中、引文、表格、下划线等样式应当正确输出；
  - 居中文本用 <p align="center"> 输出，靠右用 <p align="right"> 输出；
  - 斜体用 <i> 输出；粗体用 <b> 输出，下划线用 <u> 输出；
  - 表格头仅用 <table> 输出，表格内容保留原布局，跨页表格应注意合并；
  - 各层级标题不要加居中的 style 或 align，有换行时用 <br> 换行，多行标题（如序号、作者名等）应保证输出到一个标签中；
  - 图片可留空标签（如有附释可以加入alt），如图片为整页且将完整段落分隔时，应将图片标签放在就近段落末尾；
  - 全面复现作者批注、标记的所有效果。如某一片段侧边有横线、有打叉符号等各种标记，务必用 tag 内的纯 style 还原，不要写全局 css 类；
  - 页面的其他排版样式务必编写标签或标签的 style 属性来还原，例如：整段内容有缩进，则编写相应 margin 还原缩进效果，某部分有分栏，也应按图构造表格等效果还原，同样不要写任何全局 css 类；
  - 记录本篇目所用的主要语言，在html的lang头中填入对应语言。德语用 de，英语用 en，法语用 fr，西班牙语用 es，俄语用 ru 等。
- 7. 转换后或推理结束前务必再次检查待输出结果，根据语义校订转换中的错漏，如段前无缩进未还原、斜体未标注、页脚脚注标注错误、格式识别错误等，校对无误后再输出。
